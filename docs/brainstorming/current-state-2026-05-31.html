🚨 差距清单 — 按优先级

  • 🔴 GAP-1: 4 个 API 路由仍是 mock
    knowledge, workflow, resources, files 全部返回硬编码假数据。
    影响：前端所有 BioAgent 面板显示假数据，用户看不到真实分析结果。
  • 🔴 GAP-2: 端到端从未验证
    整个链路"用户提问→Agent推理→Tool调用→Docker执行→结果返回→前端渲染"从未完整跑过。
    影响：不知道项目是否真的能 work。
  • 🟡 GAP-3: ChromaDB / KuzuDB 未对接
    知识库的向量搜索和图谱查询层从未在运行时使用，只有本地 Wiki 文件搜索在工作。
    影响：bio_kb_query 工具只能搜 .md 文件，没有语义搜索能力。
  • 🟡 GAP-4: UI 未启动验证
    pnpm dev 能否成功？bioagent 面板是否正确渲染？GSAP 动画是否工作？未知。
    影响：可能藏着启动错误、SSR 问题、import 路径问题。
  • 🟡 GAP-5: Skill 执行未串联 Docker
    bio_skill_invoke 返回命令字符串让 Agent 再调 docker_exec，不是直接执行。
    workflow_run 只查 WorkflowDef 不实际启动引擎。
    影响：Skill 和 Workflow 是"死代码"——定义存在但不执行。
  • 🟢 GAP-6: GSAP 动画测试失败
    16 个 QCReportCard 测试因 jsdom 不支持 matchMedia 而崩溃。
    影响：CI 会红，需要 mock matchMedia。
  • 🟢 GAP-7: ui.old/ 冗余目录
    packages/ui.old 是旧版 UI，占空间。
    影响：无功能影响，但混乱。
  • 🟢 GAP-8: api/auth/* 和 api/skills/* 是 pi-web 遗留
    这些路由属于 pi-web 原生功能，BioAgent 可能不需要全部。
    影响：API 路由混乱，但功能上不碍事。
